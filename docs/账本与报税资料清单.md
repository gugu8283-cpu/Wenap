# Wenap 账本与报税资料清单（日本 · 个人/副业 SaaS）

> 备忘用，非税理士意见。有收入后建议咨询税理士一次。  
> 产品：**Wenap** · **https://wenap.app** · 收款：**Stripe（日本账户）**

---

## 1. 你需要账本吗？

**要。** 不必一开始就上复杂会计软件，但要有 **可追蹤的记录**：

| 最低配置 | 适合阶段 |
|----------|----------|
| **Google 表格 / Excel** 两张表（收入 + 经费） | 刚起步、月入还少 |
| **freee / マネーフォワード** 等 | 收入稳定、想自动生成申告草稿 |
| **税理士 + 软件** | 销售额上去、法人化、消费税登记后 |

**目的：** 确定申告时能说清「赚了多少、花了多少、客户在哪」。

---

## 2. 要不要分开「日本 / 外国」用户？

**要分开记（至少在收入表里有一列「客户国家」）。**

原因（简化）：

| 客户类型 | 和日本税的关系 |
|----------|----------------|
| **日本国内** 个人订阅 | 可能涉及 **日本消费税 10%**（你若成为课税事业者） |
| **日本国外** 订阅 | 多按 **对外提供数字服务**，日本消费税常为 **0%**（要税理士确认） |
| **欧盟等** | 可能另有 **当地 VAT**（你选 Stripe 右边 = 自己留意门槛） |

不是要你给用户体验分两站，而是 **账本 / Stripe 导出里能区分国家**。

## 3. 推荐账本结构（Google 表两张）

### 表 A：收入（每笔 Stripe 入账）

| 列名 | 填什么 | 从哪里拿 |
|------|--------|----------|
| 日期 | 付款日（UTC 或 JST 统一一种） | Stripe → 支払い / Payments |
| 种类 | `订阅初回` / `订阅续费` / `退款` | Stripe |
| 方案 | `pro` / `pro_plus` | Stripe metadata 或 金额 $9.99 / $19.99 |
| **客户国家** | `JP` / `US` / `SG` … | Stripe 客户或卡国 · **最重要** |
| 币种 | USD | 固定 |
| 税前金额 | $9.99 等 | Stripe |
| Stripe 手续费 | 自动扣的 fee | Stripe balance transaction |
| **入账单日元** | 按到账日汇率换算 | 自己用国税厅基准汇率或税理士指定方式 |
| Stripe Payment/Invoice ID | `pi_` / `in_` | 对账用 |
| 邮箱（可选） | 用户邮箱 | Stripe Customer |

**每月末：** 加一行「合计」：日本客户收入 / 海外客户收入 / 手续费合计。

### 表 B：经费（每笔支出）

| 列名 | 例子 |
|------|------|
| 日期 | 2026-05-21 |
| 内容 | Render Hosting / OpenRouter / wenap.app 域名 |
| 金额 | USD 或 JPY |
| 日元换算 | 同上 |
| 支付方式 | 信用卡 / 银行 |
| **凭证** | 发票 PDF、Stripe 发票、截图文件夹路径 |

**常见可扣经费（保留发票）：**

- Render、OpenRouter、Alpha Vantage（若有付费）
- 域名 Cloudflare / wenap.app
- Stripe 手续费（也可从收入表扣）
- PC、网络（按规则比例）
- 税理士费

## 4. 每月例行（15～20 分钟）

1. **Stripe Dashboard（本番）**  
   - **余额** → 导出 **Balance transactions**（CSV）  
   - **客户** → 可看国家（或付款里 billing country）  
   - **订阅** → 活跃数、MRR 截图存档（可选）

2. **把 CSV 重要列贴进「表 A」**（或整表导入 Google 表）

3. **经费** → 当月信用卡账单 + 服务商发票放进文件夹 `经费/2026-05/`

4. **记下当月：**  
   - 日本客户订阅笔数 / 金额  
   - 海外客户订阅笔数 / 金额  
   - 净收入（日元粗算即可）

---

## 5. 每年要留的「报税资料包」

交给税理士或自己 e-Tax 时，准备一个文件夹 **`Wenap_2025年度_申告`**（例）：

| 资料 | 说明 |
|------|------|
| **Stripe 年度报告** | Dashboard → Reports / または全年 CSV 导出 |
| **日本银行账户流水** | 显示 Stripe 入金 |
| **表 A + 表 B 全年合计** | 总收入、总经费、纯利润粗算 |
| **日本 vs 海外 汇总表** | 一页纸：日本销售额、海外销售额（日元） |
| **经费发票夹** | PDF/截图，按类别分 |
| **域名/服务器合同** | 业务相关证明 |
| **确定申告** 若用副业 | 源泉票（如有工资）、マイナンバー等 |

## 6. 日本 vs 海外 — 在 Stripe 里怎么查国家

| 方法 | 操作 |
|------|------|
| 单笔付款 | **支払い** → 点一笔 → Customer / カードの国 |
| 批量 | **Reports** → 导出 payments；看 `country` 列（若有） |
| 客户列表 | **顧客** → 地址 / 国 |

没有国别时：用 **账单地址** 或注册时 IP（较弱）；长期可在产品里记 `用户偏好国家`（非必须）。

## 7. 你可能还不知道、但以后会碰到

| 主题 | 什么时候 | 做什么 |
|------|----------|--------|
| **确定申告** | 每年 2/16～3/15 左右 | 报前年收入；副业 >20 万规则见税务 PDF |
| **消费税登记** | 销售额接近 **1000 万日元/年** | 提前问税理士 |
| **インボイス** | 成为课税事业者后 | 对日本 B2B 开发票 |
| **欧盟 VAT OSS** | 对欧盟 B2C 约 **1 万欧元/年** | Stripe Tax 或登记 OSS |
| **预扣税** | 雇人、付稿费 | Wenap 一般暂无 |
| **法人化** | 利润稳定想限责 | 设立 KK + 法人申告 |
| **Stripe 入金暂停** | KYC 未过 | 钱在 Stripe 余额；账本仍记 |

## 8. 不用现在就做、但表头可留好的

- 用户 ID（内部 UUID）
- 是否退款 /  chargeback
- 订阅开始日 / 取消日（Stripe Subscriptions 导出）
- 推荐人 / 优惠（若以后有）

---

## 9. 和 Wenap 代码的关系

- 网站 **不必** 按国家分站。  
- **报税用** 的分国别 = **Stripe + 你自己的表**。  
- 以后若要在后台自动统计「日本用户占比」，可再加管理报表（非报税必需）。

## 10. 税理士第一次见面带什么

1. 本 Markdown / `wenap-tax-guide.pdf`  
2. Stripe **过去 3 个月** CSV + 余额截图  
3. 说明：副业 SaaS、美元定价、客户全球、Stripe 日本、选右边无 +3.5%  
4. 问三句：  
   - 要不要消费税登记？  
   - 日本/海外收入怎么分？  
   - 确定申告用现金收付还是发生制？
