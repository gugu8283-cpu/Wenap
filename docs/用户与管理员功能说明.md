## 一、未登录访客

| 功能 | 说明 |
|------|------|
| 首页 `/` | 产品介绍、定价入口、公开准确率、样本入口等 |
| `/pricing` | 查看价格；若已配置 Stripe 可走结账，否则为降级提示 |
| `/accuracy` | 公开预测准确率等统计 |
| `/sample/:ticker` | 固定一批标的的**公开只读**样本报告 |
| `/about`、`/methodology` | 产品与方法论说明 |
| `/login`、`/register`、`/verify-email`、`/forgot-password` | 登录、注册、邮箱验证、找回密码 |

访客**不能**：调用需登录的 `/analyze`、自选股、个人历史等。

## 二、免费档（`free`）登录用户

### 额度与门槛

- **每月分析次数**：默认 **5 次/月**（UTC 月重置；开发可用 `WENAP_FREE_UNLIMITED` 放开）。
- **须验证邮箱**后才能发起分析。
- 存在基于 **IP** 的注册/分析频率限制（防刷）。

### 模型与报告可见范围

- 主模型默认 **Gemini 2.5 Flash Lite**（可由环境变量覆盖）。
- 返回前端的报告经 **`stripDataForViz`** 等处理：**隐藏** Pro/Pro+ 专属块（如操作建议、事件、内部人、同业对比等精简展示）。
- **来源条数**：报告中展示 **5 条**（Pro/Pro+ 为 8 条）。

### 自选与历史

- **自选股上限**：**3 只**。
- **历史**：仅**第 1 条**可完整查看；其余为锁定状态（引导升级）。
- **`/market/snapshot`**：自选批量行情快照（需后端配置 Alpha Vantage 等数据源）。

### 其它

- 头部：语言、主题、登出；免费用户可见 **QuotaStrip**（剩余次数 / 前往定价）。
- **通知中心** `/notifications`：列表、单条已读、全部已读。

## 三、Pro 档（`pro`）

### 额度

- **无 Pro+ 的「每日上限」**逻辑。
- 月额度在逻辑上按**付费无限**处理（具体以 `checkUserCanAnalyze` 为准）。

### 模型

- 默认主模型与免费档相同：**Gemini 2.5 Flash Lite**（`OPENROUTER_PRO_MODEL` 可改）。

### 报告差异（相对免费）

- 服务端会将 **同业/板块一句话** 合并进六维分析（`mergePeerLineIntoDimensions`）。
- 前端快照中保留更多 **Pro 级字段**（操作建议对象、事件、内部人摘要、同业行等）。
- 仍会弱化/剥离偏 **Pro+** 的展示：如多空对撞、情景中的触发价/时间窗、产业链长文 `analysis` 等（见 `stripDataForViz` 的 `pro` 分支）。

### 自选与历史

- **自选股上限**：**50 只**。
- **历史**：**全部**可浏览。

## 四、Pro+ 档（`pro_plus`）

### 额度

- **每个 UTC 自然日** 最多 **`WENAP_PRO_PLUS_DAILY_CAP`** 次分析（默认 **80**，可改环境变量）。
- 超出返回 `PRO_PLUS_DAILY_CAP`，提示次日 UTC 再试。

### 模型

- 默认主模型：**Claude Haiku 4.5**（`OPENROUTER_PRO_PLUS_MODEL` 可改）。
- **`secondPassCritique`**：主报告后的「批评家」第二遍（实现上与 Pro+ 主模型配置一致）。

### 报告与功能（相对 Pro）

- 快照保留 Pro+ 内容：多空对撞、情景触发价/时间窗、产业链 `analysis`、`secondPassCritique` 等。
- **PDF 导出**（打印式导出）。
- **对比页 `/compare`**：两个标的各跑一遍分析并排展示（需正确消费 SSE 流）。

### 自选

- **自选股上限**：**200 只**。

## 六、管理员（`/admin` + `/admin-api`）

### 进入方式

- 浏览器访问 **`/admin`**。
- 使用环境变量 **`ADMIN_SECRET`** 作为 **Bearer Token** 登录（与普通用户 JWT 分离）。
- 前端请求 **`/api/admin-api/...`**（开发时由 Vite 代理到后端 `/admin-api`）。

### 后台页面（前端路由）

| 页面 | 能力概要 |
|------|----------|
| Overview | 总览指标 |
| Predictions | 预测列表、筛选、准确度相关 |
| Users | 用户列表、详情、改 tier、重置试用、封禁 |
| Analysis logs | 分析日志（模型、成本、状态等） |
| Revenue | 收入统计 |
| System | API 健康、成本曲线等 |

### 后端补充（不一定每项都有独立 UI）

- **`GET /admin-api/stats/funnel`**：注册→验证→分析→付费等漏斗数据。
- 预测相关：**手动验证**、**跳过** 等（见 `routes/admin.cjs`）。

管理员**不能**代替用户完成 Stripe 扣款；支付须用户在前端完成。管理员可 **修改用户 `tier`**、查日志、做运营与风控。

## 七、环境与成本相关变量（速查）

| 变量 | 作用 |
|------|------|
| `WENAP_PRO_PLUS_DAILY_CAP` | Pro+ **每日**分析次数上限 |
| `WENAP_REFERRAL_REWARDS` | `0` / `false` / `off` 时**关闭推荐奖励发 tier**（仍可记录推荐） |
| `WENAP_CRON_SECRET` + `CRON_SERVICE_USER_ID` | 内部**每周自动分析**任务；未配置则该任务不调度 |
| `WENAP_FREE_UNLIMITED` | 开发用：免费档不限次（若项目内启用） |
