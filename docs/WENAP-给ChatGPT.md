## 一句话

**Wenap** 是一款面向个人投资者的 **AI 股票/ETF 研究报告 SaaS**：用户输入代码（如 NVDA、AAPL），系统在约一两分钟内生成一份结构化的移动端分析报告（评分、买卖信号、六维雷达、牛熊情景、来源引用等）。**不是券商，不提供下单，不构成持牌投资顾问意见**——页面有免责声明，注册/付费需勾选法律条款。

## 能做什么（用户视角）

### 所有人（含未登录）

- 浏览落地页、方法论、公开样本报告（如 `/sample/NVDA`）
- 查看历史预测准确率统计页（`/accuracy`）
- 阅读隐私政策、用户协议、免责声明（多语言）

### 免费用户（注册 + 邮箱验证后）

- 每月 **5 次** AI 分析（可配置）
- 基础报告：总分 / 信号（买·持·卖）、风险、六维雷达图、牛/基/熊情景条、走势预判、来源列表、核心结论等
- 报告为 **暗色移动端** 排版，支持中/英/日/韩/德等界面语言

### Pro（$9.99/月，Stripe 订阅）

- **不限次数** 分析（仍受合理风控）
- 在报告里额外看到：**操作建议 / 止损 / 催化剂**、**关键时间节点表**、内部人动态摘要、同行对标一行等
- 模型与免费档相同（默认偏快的 Gemini Flash Lite 类）

### Pro+（$19.99/月）

- 更强模型（默认 OpenRouter 上的 `openai/gpt-5.4-mini`），每日分析次数有上限（如 80 次/UTC 日）
- **多空对撞**（看多/看空各多条带权重）、**批评家第二轮复盘**（指出分析盲区）
- 情景可带触发价与时间窗口；产业链关联标的可有更深说明
- **导出 PDF**、**双标的对比**（`/compare`）

### 运营方（管理员）

- `/admin` 后台：用户、收入、分析日志、预测核验、账本/费用导出等（需 `ADMIN_SECRET`）

## 怎么做到的（技术流程，简化版）

```
用户浏览器
  → 登录后 JWT
  → POST /api/analyze { 代码, 期限1m~2y, 股票/ETF, 语言, 可选风险关注点 }
```

**服务端一条流水线（`server.cjs`）：**

1. **鉴权与配额**  
   检查邮箱已验证、订阅档位、免费次数 / Pro+ 日 cap。

2. **Alpha Vantage（行情与基本面）**  
   拉取 `GLOBAL_QUOTE` + `OVERVIEW`（必要时用日 K 回补价格）。  
   **现价只认这一条链路**，分析全程锁定为 `currentPrice`，并记录「价格截至 日期 时间（Alpha Vantage）」；过旧数据会在报告里打警告。

3. **数据完整性**  
   若几乎拿不到价格 → 直接报错；缺字段过多 → 可先弹窗让用户确认是否继续。

4. **大模型生成（OpenRouter）**  
   - 带联网检索的 prompt，要求输出 **严格 JSON**（分数、六维、情景、来源、结论等）。  
   - 中文/英文等各有一套 prompt；强调：事实不重复、来源要带日期、现价不得乱写。  
   - 失败时有模型降级链；Pro+ 可再跑一轮「批评家」补刀。

5. **后处理（一堆 `lib/*.cjs`）**  
   - 解析 JSON、校验目标价与情景区间是否自洽（`priceSanity`）  
   - 政策法规维不足时可补一刀  
   - 时间口径对齐到行情交易日  
   - 同一标的 1 小时内可命中缓存，避免重复烧模型额度  

6. **落库与展示**  
   - SQLite 记用户、订阅、分析日志、预测记录（日后用真实行情核验对错）  
   - SSE 流式把报告推到前端  
   - React 组件拼成卡片：Hero 分数环、雷达图、情景图、表格化结论、技术位条形图等  

**支付：** Stripe Checkout / Customer Portal + Webhook 更新用户 `tier`。  
**部署：** 前端 Vite 构建后由 Express 托管；生产在 Render（新加坡），域名 `wenap.app`。

## 技术栈一览

| 部分 | 技术 |
|------|------|
| 前端 | React 19、Vite、Tailwind、i18next |
| 后端 | Node、Express 5、单文件主入口 `server.cjs` |
| 数据库 | SQLite（`better-sqlite3`） |
| AI | OpenRouter（多模型） |
| 行情 | Alpha Vantage API |
| 邮件 | Resend（或 SMTP） |
| 支付 | Stripe（日本 Live 账户，税务自行申报） |

本地开发：`npm run dev:full`（Vite 5173 代理 API 到 3002）。

## 明确「不是什么」

- 不是证券公司、不能交易下单  
- 不是律师审过的合规投顾产品（法律文案为模板 + 用户勾选同意）  
- 没有原生 App（Web + 可选 PWA）  
- 没有 Google/GitHub 一键登录（仅邮箱注册）  
- 行情有延迟/缺失时，报告会标注数据截至日与警告，不保证实时

## 给 ChatGPT 协作时的提示

若你要帮改 Wenap 代码或写文案，请优先遵守：

1. 现价、技术位、情景区间必须以 **Alpha Vantage 锁定价** 为基准，不要让模型或前端 second-fetch 覆盖。  
2. 改法律/订阅相关时，需同步 `lib/legalVersions.cjs` 版本号，老用户会进 `/accept-legal`。  
3. Stripe 环境变量用 `STRIPE_PRICE_PRO` / `STRIPE_PRICE_PRO_PLUS`，不是单一的 `STRIPE_PRICE`。  
4. 产品报告 UI 在瘦身：多用表格/要点，**保留**雷达图与情景图；细节见 `WENAP-CLAUDE-CONTEXT.md` 第 19 节。  
5. 不要提交 `.env` 里的密钥。
